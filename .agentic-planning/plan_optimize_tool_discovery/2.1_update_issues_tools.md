# Этап 2.1: Обновить Issues Tools с метаданными

**Цель:** Добавить category, subcategory, priority, tags во все инструменты категории Issues

**Затронутые файлы:** 8 tools в `packages/servers/yandex-tracker/src/tools/api/issues/`

**Время:** 2-2.5 часа

**Зависимости:** Требует завершения этапа 1.1

**Параллельность:** Можно выполнять параллельно с этапом 2.2

## Категоризация Issues Tools

### Read Operations (subcategory: "read")

| Tool | Priority | Tags |
|------|----------|------|
| `GetIssuesTool` | `critical` | `["read", "get", "fetch", "issue"]` |
| `FindIssuesTool` | `critical` | `["search", "query", "filter", "issues"]` |
| `GetIssueChangelogTool` | `high` | `["history", "changelog", "audit", "read"]` |

| Tool | Priority | Tags |
|------|----------|------|
| `CreateIssueTool` | `critical` | `["create", "new", "write", "issue"]` |

| Tool | Priority | Tags |
|------|----------|------|
